# Utilities und Funktionen

<ViewSource path="lib" />

Unter dem Endpoint `@northware/ui/lib` werden unterstützende Funktionen bereitgestellt, die in den Nothware Packages und Apps an mehreren Stellen verwendet werden.

## Fonts

Die Northware Apps verwenden die Schriftart [Source Sans 3](https://fonts.google.com/specimen/Source+Sans+3) und die Schriftart [Source Code Pro](https://fonts.google.com/specimen/Source+Code+Pro) von Google Fonts. 
Die Schriftarten müssen über die `layout.tsx` der App importiert werden und es braucht dafür die Definition einer Variable.
Über die Variable `fonts` werden die beiden Schriftarten geladen und es werden die Klassen `.font-sans` und `.antialiased` auf das HTML-Attribut angewendet.
Für die Verwendung der Schriftart Source Code Pro für einzelne Bereiche, reicht es aus, `.font-mono` auf einzelne Attribute anzuwenden.

- [Mehr zu der Nutzung in einer App](?path=/docs/northware-ui-verwendung-in-einer-app--docs#die-layouttsx)
- [Über die Verwendung von Fonts in Next.js mit Tailwind CSS](https://nextjs.org/docs/app/building-your-application/optimizing/fonts#with-tailwind-css)

## `cn` Helper

Die Funktion `cn` wird unter `@northware/ui/lib/utils` zur Verfügung gestellt. 
Hierbei handelt es sich um eine Helper-Function, die steuert, das CSS-Klassen, die auf eine einzelne Instanz einer Komponente angewendet werden gegenüber Klassen, 
die innerhalb der Komponente definiert wurden bevorzugt werden. So lassen sich die Komponenten auf eine andere als die vordefinierte Weise stylen, 
auch wenn dieses Styling, dem vordefinierten Styling entgegensteht.

Um dies zu ermöglichen werden de Tools [`clsx`](https://github.com/lukeed/clsx) und [`tailwind-merge`](https://github.com/dcastil/tailwind-merge) verwendet.
Dieses Vorgehen wird von shadcn/ui vorgeschlagen und die Komponenten sind auf die Verwendung der Funktion `cn` angewiesen. [Mehr dazu](https://ui.shadcn.com/docs/installation/manual#add-a-cn-helper)

## `toast` Funktion von `sonner`

<ShadcnDocs path="components/sonner" />

Mithilfe der `toast()` Funktion kann ein Event-Feedback gezeigt werden. Die Funktion kann über `@northware/ui/lib/utils` importiert werden. 
So muss nicht in jeder App die Dependency `sonner` installiert werden.
